# 1. Github

github.com is a site that offer services based on git. It has many, many features to help developers create, manage, share, debug and deploy their projects. Github has a free tier that allows you to create an account, store (privately or publicly) your projects and share them with your peers.

Login to github.com and, on the top left side click on the button to create a new repository.

You are going to create a repo to store a set of reference files that will help you in your career. Here are the steps:

  • On the repository name type awesomedev
  • Make sure the repo is public
  • Check the Add a README file option
  • Click on Create repository.

You should have something similar to this now.

  1. Menubar that gives access to many github features. We will concentrate on code only.
  2. Show which branch we are looking at. (Where HEAD is pointing to)
  3. Show how many branches this repo has.
  4. Show what was the branch's latest commit and the number of commits
  5. Show the hash of the latest commit.
  6. Show a list of the files in the repo.
  7. Show, for each file, what was the commit that changed it.
  8. The rendered README.md
  9. Edit button

Excellent. You created a github repo!

# 3. Cloning a repo

To clone the repo, go to the Code button and copy the repo link.

Then, on terminal, open a session and type:

git clone <link_that_you_copied>
1

You now have a copy of the remote repo on your computer, ready for you to work.

Activity

  1. On your new repo awesomedev click on the edit button to edit the README.md file.
  2. Add the title "Git commands I should know by heart".
  3. Preview changes.
  4. Commit README.me with the message "Add a title to this repo".
  5. Click on the edit button again
  6. Add the following commands to a new list under the title:
    • git init
    • git status
    • git log
    • git add
    • git commit
    • git checkout
    • git clone
    • git branch
    • git push
  7. Preview changes
  8. Commit README.me with the message "Add a initial list of git commnands".
  9. Create a new file on the repo and name it links.md
  10. Add a title: "Links"
  11. Add a subtitle: "Git"
  12. Add links to the Following URLs
    • https://www.atlassian.com/git/tutorials/atlassian-git-cheatsheet
    • https://education.github.com/git-cheat-sheet-education.pdf
    • https://about.gitlab.com/images/press/git-cheat-sheet.pdf
  13. Preview the changes
  14. Format the 1st link to read "Atlassian git cheatsheet"
  15. Commit with message "Adds links file with git links on it" (Is there anything wrong with this message?)
  16. Check the history (log) on Github
  17. Check the hashes
  18. Click in one hash to see the changes
  19. Go back to the main repo view
  20. Clone the repo to your machine
  21. Fix the second and third links to read Github cheatsheet and Gitlab cheatsheet respectively.
  22. Commit your changes with the message "Format links"
  23. Push your changes to the repo
  24. Check your changes.

# 4. Forking a repo

As we saw, clonning makes a exact copy of a github repo to your local machine. Sometimes though, this is not desireable or possible. In those cases a copy is made "sideways", so, instead of copying a repo to you machine, you first, copy it to another repo on your own account and then clone this repo.

Cloning

Forking

Forking and Pull Requests (PRs) are used in virtually every single Open Source project out there. If you want to participate and contribute to OS projects you should be familiarized with it. Also, many companies will require forking and PR knowledge so the senior engineer responsible for the source code can evaluate your code and add it to the main codebase.

Activity
